# 행동 기반 CRM 추천 시스템 설계

## 목표

`shopping_mall_crm_behavior_sample_1000.json`의 행동 이벤트 1,000건을 새 PostgreSQL 데이터베이스에 적재하고, 어떤 고객에게 어떤 상품·카테고리와 CRM 액션을 추천할지 보여주는 Gradio 애플리케이션을 구축한다.

기존 `app_old.py`와 `etf_database.db`는 원본 그대로 보존한다. 새 애플리케이션은 `app.py`에 작성한다.

## 아키텍처

데이터 흐름은 다음과 같다.

1. JSON의 `records` 배열을 읽는다.
2. 중첩 필드를 분석용 단일 PostgreSQL 테이블 `crm_events`로 펼친다.
3. 고객별 행동을 집계해 관심 카테고리, 상품 후보, 구매 가능성, 할인 민감도, 휴면 위험, 캠페인 반응도를 계산한다.
4. 결정론적 점수 엔진이 상품·카테고리와 CRM 액션을 추천한다.
5. 선택적으로 LLM이 추천 근거와 고객 메시지를 자연어로 다듬는다.
6. Gradio에서 개별 고객 추천, 우선 연락 고객 목록, 자연어 분석을 제공한다.

LLM은 추천 점수를 결정하지 않는다. API 키가 없거나 호출에 실패해도 규칙 기반 추천은 동작해야 한다.

## 데이터베이스

Docker Compose로 PostgreSQL을 실행한다. 애플리케이션과 적재 스크립트는 `DATABASE_URL` 환경 변수로 접속한다.

`crm_events`는 JSON 이벤트 한 건을 한 행으로 저장한다. 다음 영역의 필드를 명시적 PostgreSQL 타입으로 펼친다.

- 이벤트: 이벤트 ID, 발생 시각, 이벤트 유형, 체류 시간, 검색어
- 고객: 고객 ID, 연령대, 성별, 지역, 회원 등급, 신규 여부
- 세션: 세션 ID, 채널, 유입 경로, 기기 유형
- 상품: 상품 ID, 상품명, 카테고리, 가격, 할인율, 할인가
- 장바구니·주문: 장바구니 상품 수, 수량, 주문 ID, 주문 금액, 구매 여부
- 캠페인: 캠페인명, 쿠폰 사용·다운로드 여부
- CRM: 이전 구매 횟수, 마지막 구매 후 경과일, 태그 배열, 추정 세그먼트

`event_id`를 기본 키로 사용하고 고객, 시각, 이벤트 유형, 상품, 카테고리, 구매 여부에 분석용 인덱스를 둔다. 적재는 `event_id` 충돌 시 갱신하는 upsert 방식으로 만들어 반복 실행해도 정확히 1,000행을 유지한다.

## 추천 엔진

고객과 상품·카테고리 후보별 기본 행동 점수는 다음과 같다.

- 구매: 5점
- 장바구니 추가 또는 양수 수량: 4점
- 찜하기: 3점
- 상품 조회: 2점
- 검색: 2점
- 쿠폰 다운로드: 할인 상품에 2점

최근 행동일수록 가중치를 높이고, 같은 카테고리의 반복 행동과 고객의 관찰된 선호 가격대 일치에 가점을 준다. 이미 구매한 상품은 반복 구매 성격이 확인되지 않으면 감점한다. 할인 민감 태그 또는 쿠폰 반응이 있으면 할인율이 높은 상품을 우선하고, 할인 민감도가 낮은 우수 고객에게는 신상품·프리미엄 후보를 우선한다.

후보 점수는 고객 내에서 0~100으로 정규화하고 상위 3개를 반환한다. 데이터가 부족한 고객은 같은 세그먼트·회원 등급에서 인기 있는 카테고리와 상품을 fallback으로 사용한다.

CRM 액션은 다음 우선순위 규칙으로 결정한다.

- 장바구니 후 미구매: 장바구니 이탈 리마인드와 관련 상품 쿠폰
- 쿠폰 다운로드 후 미구매: 쿠폰 만료 알림
- 마지막 구매 후 경과일이 길거나 휴면 위험 태그: 재활성화 쿠폰
- 신규 고객: 첫 구매 혜택
- 구매 이력이 많은 GOLD/VIP 고객: 재구매 또는 교차판매
- 캠페인 반응 고객: 관찰된 반응 채널 중심 푸시
- 할인 민감 고객: 할인 상품과 쿠폰 중심 제안
- 우수 고객이면서 할인 민감도가 낮음: 신상품·프리미엄 제안

각 추천은 고객 ID, 추천 상품·카테고리, 0~100 점수, CRM 액션, 채널, 행동 근거 목록과 메시지를 포함한다.

## Gradio 애플리케이션

새 `app.py`는 다음 기능을 제공한다.

1. 고객 ID 또는 세그먼트·등급·지역 필터로 고객을 선택한다.
2. 최근 활동, 구매 횟수, 관심 카테고리, 할인 및 캠페인 반응을 요약한다.
3. 추천 상품·카테고리 최대 3개와 점수, CRM 액션, 채널, 행동 근거를 표시한다.
4. 규칙 기반 템플릿 또는 선택적 LLM으로 고객 발송용 한국어 메시지를 표시한다.
5. 전체 고객 중 우선 연락할 대상을 추천 우선순위로 정렬한다.
6. 자연어 질문을 읽기 전용 SQL로 변환해 분석 결과와 사용한 SQL을 보여준다.

자연어 SQL은 단일 `SELECT` 또는 `WITH ... SELECT`만 허용하고, 다중 문장 및 데이터 변경·DDL 키워드는 거부한다.

## 오류 처리와 운영

- PostgreSQL 연결 실패 시 자격 증명을 노출하지 않고 실행 방법을 안내한다.
- JSON 구조 또는 필수 필드가 잘못되면 적재를 중단하고 문제 이벤트를 식별한다.
- OpenAI 설정이 없거나 호출이 실패하면 결정론적 메시지 템플릿을 사용한다.
- 비밀번호와 API 키는 커밋하지 않으며 `.env.example`만 제공한다.
- Docker healthcheck를 사용해 PostgreSQL 준비 상태를 확인한다.

## 검증

- Compose 설정 유효성 검사
- PostgreSQL healthcheck 통과
- 스키마 생성 및 JSON 1,000행 적재 확인
- 적재를 두 번 실행해도 1,000행인 멱등성 확인
- 필수 컬럼 타입, 기본 키, 인덱스 확인
- 대표 고객 유형별 추천 규칙 단위 테스트
- LLM 없이 추천 및 메시지 생성 가능 여부 확인
- 자연어 SQL 읽기 전용 검증 테스트
- `app.py` import 및 Python 컴파일 확인
- `app_old.py`와 `etf_database.db`의 작업 전후 해시 동일성 확인
